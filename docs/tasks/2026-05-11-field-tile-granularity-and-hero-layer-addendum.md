# Field Tile Granularity & Hero Layer Addendum

## 目的

`2026-05-11-field-battle-visual-reference-and-modern-hero.md` の補足タスク。

ユーザー要望として、フィールド画面について以下を追加で精緻化する。

1. タイルをもっと細かく見せる。
2. 主人公キャラは現在地そのものを置き換えるのではなく、現在地タイルの上に乗っているように見せる。
3. フィールドを「25個の大きなマス」ではなく、小さな地形が連続したRPGフィールドとして見せる。

この補足は、実装時には本体タスクファイルと合わせて必ず読むこと。

---

## 1. 重要な考え方

### 1.1 論理マップと見た目マップを分ける

現在のプロトタイプでは、移動・エンカウント・宝箱・ボス判定に使う論理マップが存在している。

今回の改善では、必ずしもゲームロジックを大きく変える必要はない。
ただし、見た目はより細かいタイル密度にする。

方針：

- ロジック用のマップは既存のままでもよい。
- 表示用には、各マスをさらに細かいサブタイルで描画する。
- 可能なら、論理マップ自体も 5x5 から 9x9 以上へ拡張する。
- ただし、既存のクエスト進行・宝箱・ボス・町戻りが壊れるなら、まずは表示粒度だけ上げる。

優先順位：

```txt
P0: 5x5 論理マップを維持しつつ、1マス内を細かく描画する
P1: 表示上は 9x9 以上に見えるようにする
P2: 論理マップ自体を 9x9 / 11x11 へ拡張する
```

---

## 2. タイル粒度の要件

### 2.1 避けたい状態

以下のような状態は避ける。

- 5x5 の大きな色ブロックに見える。
- 各タイルが単色に近い。
- 町・草原・森・山・水辺がただの色違いに見える。
- 主人公がタイルそのものの文字やラベルのように見える。
- フィールドというより管理画面のグリッドに見える。

### 2.2 目指す状態

以下のように見せる。

- 1つの論理タイル内に、細かい地形パターンがある。
- 草原は小さな草の粒がある。
- 森は木が密集して見える。
- 山は小さな山型が複数ある。
- 水辺は波や濃淡がある。
- 道は隣接タイルへ自然につながる。
- 町・宝箱・ボス地点は、地形の上にオブジェクトとして乗っている。
- 主人公は現在地タイルの上に立っている。

## 3. 実装パターン

### 3.1 P0: サブタイル描画

論理タイル1つを、視覚的には 4x4 または 8x8 のサブタイルで表現する。

例：

```txt
論理タイル: grass
表示:
[草][草][草][小石]
[草][明][草][草]
[草][草][花][草]
[影][草][草][草]
```

実装候補：

- CSS `background-size` を小さくする。
- `repeating-linear-gradient` を使う。
- `radial-gradient` を複数重ねる。
- `::before` / `::after` で小物を置く。
- 必要であれば `FieldTile` 内で `miniTiles` を描画する。

### 3.2 P1: 見た目だけ 9x9 以上に見せる

既存の 5x5 論理マップを維持しつつ、表示上はより広いマップに見せる。

方法例：

```txt
logical fieldMap: 5x5
visual tile density: 10x10相当
```

各論理タイルを 2x2 の表示タイルに展開すると、見た目は 10x10 相当になる。

この場合：

- 移動判定は既存のまま。
- 表示密度だけ上がる。
- 主人公は該当論理タイルの中央に乗る。
- 宝箱・町・ボスも該当論理タイルの中央または少し奥に配置する。

### 3.3 P2: 論理マップの拡張

余力がある場合、論理マップ自体を 9x9 または 11x11 に拡張する。

ただし、P2でやる場合は以下を守る。

- 町の初期位置を分かりやすくする。
- 町から草原、森、宝箱、ボスまでの導線を維持する。
- エンカウント頻度を再調整する。
- 宝箱取得フラグが壊れないようにする。
- ボス地点に到達できることを確認する。
- 水や山で詰まないようにする。

P0実装では、マップ拡張は必須ではない。

## 4. タイルレイヤー設計

フィールドタイルは、以下のレイヤーに分けて考える。

```txt
[Base Terrain Layer]
  grass / forest / road / water / hill / shore など

[Detail Layer]
  草粒 / 木の密度 / 波 / 山影 / 道の縁 / 小石 など

[Object Layer]
  町 / 宝箱 / ボス地点 / 光 / 看板 など

[Hero Layer]
  現在地に立つ主人公

[Selection / Shadow Layer]
  主人公の足元影、現在地の軽いハイライト
```

重要：

- 主人公は `Base Terrain` を置き換えない。
- 主人公は `Object Layer` より上、UIより下に置く。
- 現在地タイルの地形は主人公の下に見えていること。
- 宝箱やボス地点と同じタイルにいる場合でも、重なりが破綻しないこと。

## 5. 主人公の現在地表現

### 5.1 目標

主人公キャラは「現在地の上に乗っている」ように見せる。

避けたい表現：

- タイル内に「勇」という文字が置かれているだけ。
- タイル全体が主人公色になる。
- 主人公が背景と一体化している。
- 現在地タイルの地形が消える。

目指す表現：

- 主人公の足元に影がある。
- タイルの中央やや下に立っている。
- 地形の上にキャラが重なって見える。
- 主人公の下に現在地タイルが見える。
- 主人公は地形や宝箱より前面に見える。

### 5.2 HeroSprite配置ルール

`HeroSprite` は、タイル内の通常コンテンツではなく、オーバーレイとして配置する。

推奨：

```tsx
<div className="field-map">
  <div className="terrain-grid">
    {tiles.map(tile => <FieldTile tile={tile} />)}
  </div>
  <HeroSprite position={game.position} />
</div>
```

または、タイル内で描画する場合も、必ず絶対配置にする。

```tsx
<div className="field-tile">
  <TerrainLayer tile={tile} />
  <ObjectLayer tile={tile} />
  {playerHere ? <HeroSprite /> : null}
</div>
```

### 5.3 CSS目安

```css
.field-map {
  position: relative;
}

.field-tile {
  position: relative;
  overflow: visible;
}

.hero-sprite {
  position: absolute;
  left: 50%;
  bottom: 12%;
  transform: translateX(-50%);
  z-index: 20;
  pointer-events: none;
}

.hero-shadow {
  position: absolute;
  left: 50%;
  bottom: 8%;
  transform: translateX(-50%);
  width: 48%;
  height: 14%;
  border-radius: 999px;
  background: rgba(0, 0, 0, 0.28);
  z-index: 19;
}
```

### 5.4 サイズ目安

| 対象 | 目安 |
|---|---:|
| HeroSprite横幅 | タイル幅の 48〜70% |
| HeroSprite高さ | タイル高さの 65〜110% |
| 足元影 | タイル幅の 40〜55% |
| z-index | terrain < object < heroShadow < hero < UI |

主人公は少しだけタイルから上にはみ出してよい。
ただし、隣のタイルや操作ボタンを邪魔しないこと。

## 6. オブジェクトと主人公の重なり

### 6.1 宝箱タイル

宝箱タイルに主人公が乗った場合：

- 宝箱は主人公の背後または横に見える。
- 取得済みなら空箱や薄い表示にする。
- 主人公が宝箱を完全に隠してもよいが、ログで取得結果が分かること。

### 6.2 町タイル

町タイルに主人公が乗った場合：

- 町アイコンや屋根の上に主人公が立つようには見せすぎない。
- 町の入口・道の上にいるような配置が望ましい。
- 町タイル内では、主人公を少し下寄せに配置する。

### 6.3 ボスタイル

ボスタイルに主人公が乗った場合：

- 戦闘開始前に危険地点へ踏み込んだ印象が出る。
- 主人公の足元影と、ボス地点の暗い気配が重なっても視認できること。
- ボス撃破後は、同じタイルが「光」または「静かな跡地」に変わるとよい。

## 7. タイル別の細密表現仕様

### 7.1 grass

- 明るい緑をベースにする。
- 小さな草粒や色ムラを入れる。
- 安全地帯に近い印象。
- まばらに花や小石のような点を置いてもよい。

### 7.2 forest

- 濃い緑をベースにする。
- 木の集合に見える丸い影や三角形を入れる。
- grassより密度を高くする。
- 危険感を少し出す。

### 7.3 road

- 茶色・土色ベース。
- 隣接する road / town / boss 方向へつながって見えること。
- 可能なら、隣接判定で道の向きを変える。
- 難しければ、中央に道筋を通すだけでもよい。

### 7.4 water

- 青の濃淡で波を出す。
- 通れないタイルと分かるようにする。
- 反射や斜線パターンを入れてもよい。

### 7.5 shore

- 砂色と水色の境界。
- 水辺と陸の接続に見えること。
- 通れるか通れないかが分かりにくくならないようにする。

### 7.6 hill

- 山や丘の立体感を出す。
- 小さな三角形、斜面、影を使う。
- 通れないまたは進みにくい地形に見えること。

### 7.7 town

- 町・拠点として見えること。
- 小さな屋根、入口、道との接続を表現する。
- 安全色を使う。
- 文字だけの「町」に頼りすぎない。

### 7.8 chest

- 地形の上に宝箱オブジェクトを置く。
- 宝箱はタイルそのものではなく object layer として扱う。
- 未取得と取得済みの違いを見せる。

### 7.9 boss

- 地形の上に危険地点の気配を置く。
- 暗い円、光、影、枠などで特別感を出す。
- ボス撃破前後で表示を変えられる構造にする。

## 8. 実装受け入れ条件の追加

本補足により、以下を受け入れ条件に追加する。

### 8.1 フィールド粒度

- フィールドが 5x5 の単色ブロックに見えない。
- 各タイル内に細かい地形表現がある。
- 草原・森・道・水・山の違いが、色だけでなく質感でも分かる。
- 宝箱・町・ボスは地形の上にオブジェクトとして乗っている。

### 8.2 主人公表示

- 主人公は現在地タイルの上に乗って見える。
- 現在地タイルの地形が主人公の下に残っている。
- 主人公の足元影または現在地ハイライトがある。
- 主人公が背景に埋もれない。
- `HeroSprite` が画像未配置でもCSSアイコンで表示される。
- 「勇」テキストは最終フォールバックに留める。

### 8.3 レイヤー

- terrain / object / hero / UI の重なり順が明確。
- 宝箱タイル、町タイル、ボスタイルに主人公が乗っても表示が破綻しない。
- `overflow: hidden` によって主人公の頭や影が不自然に切れない。
