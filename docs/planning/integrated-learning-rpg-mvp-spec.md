# Learning RPG 統合MVP仕様

## 1. 目的

このドキュメントは、Learning RPG の現時点の主要仕様を統合し、Chapter 1「飛鳥の町」MVPとして実装判断できる形にまとめる。

個別仕様として検討してきた以下を、ひとつのゲーム体験として接続する。

- 史実性最優先の方針
- 史実カード収集
- カードパック / 時代タグ
- Sランクカード
- 複数収集カード
- カード3択インタラクション
- カードゲート進行
- 一枚絵探索UI
- 吹き出しホットスポット
- Chapter 1の勝利条件

## 2. ゲームの一文定義

> ストーリーで時代に入り込み、一枚絵の町を探索し、史実カードを集め、カードを使って人・物・場所の関係を読み解くカードRPG。

このゲームの目的は、歴史を暗記することではない。

プレイヤーが町を歩くように絵を眺め、人や物に触れ、史実カードを集め、それらを使って時代の構造を理解していくことを目指す。

## 3. 最重要方針

### 3.1 史実性を最優先する

Learning RPG の屋台骨は、史実的に確からしいこと。

面白さ、かわいさ、ゲーム的な分かりやすさは重要だが、史実性を損なう場合は採用しない。

### 3.2 カードは読むものではなく使うもの

カードは図鑑として眺めるだけではない。

NPCに見せる、場所で使う、イベントの手がかりにする、複数カードをつなげることで、ストーリーを進める。

### 3.3 カードを集めるゲームではなく、カードで時代を読み解くゲームにする

カード収集は目的ではなく、時代理解を進めるための手段。

ただし、カード収集自体にもゲーム的な気持ちよさを持たせる。

## 4. Chapter 1 MVPの基本体験

Chapter 1「飛鳥の町」では、プレイヤーは以下を体験する。

1. 現代の子どもである主人公が、うーちゃんと出会う
2. 飛鳥の町へ入る
3. 一枚絵の町画面を見る
4. 吹き出しホットスポットをタップして人・物・場所を調べる
5. 史実カードを獲得する
6. 手持ちカードから3択でカードを見せ、NPCやイベントの反応を見る
7. カード条件を満たして、次の場所やイベントを開く
8. 寺で起きている混乱をカードで読み解く
9. Sランクカード「飛鳥寺」を解放する

## 6. 吹き出しホットスポット

### 6.1 ホットスポットの種類

| 種類 | 内容 | 例 |
|---|---|---|
| 会話 | NPCと話す | 店の人、職人、役人、僧 |
| 観察 | 物や建物を見る | 市の荷物、供物、瓦、木簡、寺の屋根 |
| イベント | ストーリーが進む | 寺の人が来ない、職人に注意される |
| カード | カード獲得やカード使用 | お寺の瓦、木簡 |
| 入室 | サブ画面へ移動 | 作業場、寺の入口、寺内部 |

### 6.2 吹き出し状態

吹き出しには状態を持たせる。

| 状態 | 意味 | 表現方針 |
|---|---|---|
| 未確認 | まだ触れていない | 明るい色、軽い揺れ、！や？ |
| 進行中 | 一部完了、条件途中 | 中間色、小さな点滅、進捗感 |
| 完了済み | 主要イベント完了 | 落ち着いた色、チェック、小さめ表示 |
| 再訪あり | 新カードや時代リンクで追加反応あり | 星や再発見アイコン |
| ロック中 | 条件不足 | 薄い色、鍵アイコン、ヒント表示 |

### 6.3 完了済み表示

終わったところは必ず吹き出しの色や形を変える。

これにより、以下を防ぐ。

- 同じ場所を何度も押してしまう
- どこまで進んだか分からなくなる
- 未完了ポイントを見落とす
- 子どもが次の行動で迷う

## 7. 史実カード

### 7.1 カードカテゴリ

- 人物
- 建物
- 道具・技術
- 食・供物
- 服装
- 事件・制度
- 戦争・争い

### 7.2 レアリティ

| レアリティ | 意味 |
|---|---|
| S | 時代の核心。国宝・重要文化財級、象徴的カード |
| A | 章の中核カード |
| B | 生活感・背景を支えるカード |
| C | 演出候補。史実確認前は中核にしない |
| D | 不採用候補 |

### 7.3 枚数タイプ

| タイプ | 内容 | 例 |
|---|---|---|
| Unique | 1枚だけ | 飛鳥寺、聖徳太子、推古天皇、蘇我馬子 |
| Stackable | 複数枚集める | お寺の瓦、木簡、供物、市の荷物、木材 |
| Variant | 差分カード | 飛鳥の瓦、奈良の瓦、木簡、文書、瓦版 |

### 7.4 時代タグ

カードには主時代タグを付ける。

例：

- 飛鳥
- 奈良
- 平安
- 鎌倉
- 室町
- 江戸
- 明治

関連時代タグは最初から全部見せない。

後の時代で関連カードを見つけることで、過去カードに時代リンクが追加される。

例：

1. 飛鳥で「お寺の瓦」を発見：時代タグは飛鳥のみ
2. 奈良で大寺院の瓦を発見：お寺の瓦に奈良リンク追加
3. 室町で禅寺の屋根を発見：室町リンク追加

## 8. カードパック

### 8.1 パック種類

| 種類 | 内容 | 例 |
|---|---|---|
| 時代パック | 時代別カード集 | 飛鳥パック、奈良パック |
| テーマパック | 時代横断テーマ | 仏教文化、文字と記録、建物と技術 |
| 章パック | ストーリー章に対応 | Chapter 1：飛鳥の町パック |

### 8.2 総数表示

時代パックや章パックは、最初に総数を見せる。

例：

> 飛鳥パック 1 / 14

ただし、未発見カードの詳細は伏せる。

例：

> 未発見Sカード 1 / 建物  
> ヒント：町の奥に見える、大きなお寺に関係する。

### 8.3 MVP枚数

Chapter 1 MVPでは、飛鳥パックは12〜16枚に絞る。

| 種類 | 枚数目安 |
|---|---:|
| S | 1〜2 |
| A | 5〜6 |
| B | 5〜8 |
| C | 0〜2 |

## 10. カード3択インタラクション

### 10.1 基本仕様

イベントやNPC会話では、手持ちカードから3枚が提示される。

プレイヤーは「どれを選ぶ？」に対して1枚を選ぶ。

選んだカードによって、NPCや状況の反応が変わる。

### 10.2 3択の構成

| 種類 | 役割 |
|---|---|
| 本命カード | イベント進行に強く関係する |
| 関連カード | 追加会話は出るが本筋ではない |
| 少しズレたカード | 今は弱いが別ルートのヒントになる |

### 10.3 反応タイプ

#### 強反応

- イベントが進む
- 新しい情報が出る
- 関連カードがつながる
- Sカードの解放条件が進む

#### 中反応

- 追加会話が出る
- 理解度が少し上がる
- 本筋は大きく進まない

#### 弱反応

- 軽いリアクション
- 別NPCや別ルートへのヒント
- 不正解扱いではない

### 10.4 例：職人イベント

```text
職人さんに、どのカードを見せる？

[お寺の瓦]
[木材を運ぶ人々]
[供物]
```

- お寺の瓦：強反応
- 木材を運ぶ人々：中反応
- 供物：弱反応。僧ルートへのヒント

### 10.5 設計思想

> カード選択は、歴史クイズではなく、関係の強さを探す遊びである。

## 11. カードゲート進行

### 11.1 基本仕様

特定カード、複数枚カード、関連カードセットを持っていることで、新しい会話・観察・場所・イベントが開く。

### 11.2 ゲート種類

| ゲート | 内容 | 例 |
|---|---|---|
| 単体カードゲート | 特定カードが必要 | お寺の瓦で職人会話が進む |
| 複数枚ゲート | 同カードを一定枚数集める | 瓦3枚で建築理解が深まる |
| セットゲート | 関連カード群が必要 | 飛鳥寺Sカード解放 |
| カテゴリ枚数ゲート | 特定カテゴリを一定数集める | 道具・技術カード3枚 |
| 時代横断ゲート | 後の時代で過去カードが更新 | 瓦に奈良リンク追加 |

### 11.3 ハードゲートとソフトゲート

MVPではソフトゲート中心にする。

ハードゲートは、章終盤やSランク完全解放などに限定する。

### 11.4 詰み防止

- 必須カードは後から取りに戻れる
- 条件不足時はヒントを出す
- 関連カードで中反応を出し、必要な場所へ誘導する
- Sカード進捗をカード帳に表示する

## 12. Sランク「飛鳥寺」解放仕様

### 12.1 解放条件

Sランク「飛鳥寺」は、寺に到着するだけでは完全解放されない。

以下5要素を満たすことで解放する。

| 要素 | 条件例 |
|---|---|
| 建築 | お寺の瓦、または寺をつくる職人たち |
| 人 | 僧、または職人 |
| 儀式 | 供物 |
| 記録 | 木簡、または役人 |
| 場所 | 寺の入口イベント到達 |

### 12.2 解放演出

> Sランクカード「飛鳥寺」が記録帳に刻まれた。  
> 建物、職人、僧、供物、記録。いろいろなものが集まって、このお寺は形になっていた。

### 12.3 進捗UI

```text
大きなお寺のカード
手がかり 3 / 5

✓ 建物
✓ 人
✓ 儀式
□ 記録
□ 場所
```

## 13. Chapter 1のイベント進行案

### 13.1 市の一角

#### 状況

寺へ届ける荷物があるが、寺の人が取りに来ない。

#### 入手カード

- 市の荷物
- 供物

#### 3択例

```text
どのカードを見せて聞いてみる？

[市の荷物]
[供物]
[お寺の瓦]
```

#### 反応

- 市の荷物：寺へ向かう荷物だと分かる
- 供物：大事な場で使うものだと分かる
- お寺の瓦：職人に聞くとよさそう、というヒント

### 13.2 職人の作業場

#### 状況

寺へ向かう途中で、瓦や木材を運ぶ職人たちに出会う。

#### 入手カード

- お寺の瓦
- 寺をつくる職人たち
- 木材を運ぶ人々

#### 3択例

```text
職人さんに、どのカードを見せる？

[お寺の瓦]
[木材を運ぶ人々]
[供物]
```

### 13.3 役人・木簡イベント

#### 状況

役人が荷物や人の動きを確認している。

#### 入手カード

- 木簡
- 役人

#### 3択例

```text
役人さんに、どのカードを見せる？

[木簡]
[市の荷物]
[お寺の瓦]
```

### 13.4 寺の入口

#### 状況

寺では儀式準備と建設作業が重なっている。

#### 進行

カード接続パズルで、寺の混乱の理由を読み解く。

#### 終盤パズル例

```text
寺へ届くはずだったものは？

[供物]
[お寺の瓦]
[木簡]
```

```text
それを受け取るはずだった人たちは？

[僧]
[役人]
[職人]
```

```text
寺がばたばたしていた理由に関係ありそうなのは？

[寺をつくる職人たち]
[市の荷物]
[宮に仕える人]
```

#### 結果

> 寺では、儀式の準備と建設作業が重なっていた。  
> 市から届く荷、供物、職人たちの作業、僧たちの準備が一つの場所に集まっていた。

## 16. GPT image2用の一枚絵制作方針

### 16.1 基本方針

一枚絵は、ただの背景ではなく、史実カードが隠れた探索画面として作る。

### 16.2 注意点

- 史実性を優先する
- 近世・近代的な屋台や町並みを混ぜない
- 現代的な看板や道具を入れない
- スマホ縦画面で見やすい余白を残す
- 吹き出しを後から乗せやすい構図にする
- ホットスポットが見やすいように、人・物・建物を整理して配置する

### 16.3 構図

- 手前：最初に触る対象
- 中景：会話NPCや物資
- 奥：目的地やSカードに関わる対象
- 端：後から開くサブ画面への入口

## 18. 成功条件

- 一枚絵を見るだけで時代感が伝わる
- 吹き出しで次に触る場所が分かる
- 完了済みと未完了が直感的に分かる
- カードを集める意味がイベント進行につながっている
- カード3択がクイズではなく、関係の強さを探す遊びになっている
- Sカード解放が単なる報酬ではなく、時代理解の節目になっている
- 史実性が崩れていない
- 子どもでも触れる
- 大人が見てもカード関係が面白い

## 19. 失敗条件

- ただの会話ADVになる
- ただの図鑑になる
- カードが収集物で終わる
- カード3択が単なる正解当てになる
- どのカードを選んでも同じ反応になる
- 吹き出しの状態が分かりにくい
- 一枚絵に情報を詰め込みすぎる
- 完了済みの場所が分からず迷う
- 必須カードを取り逃して詰む
- 史実的に弱い演出が中核になる

## 20. 最終結論

Learning RPG のMVPは、以下を成立させる。

> 一枚絵の時代絵巻をタップして探索する。  
> 人・物・場所から史実カードを集める。  
> 手持ちカードから3択でカードを使い、NPCやイベントの反応を変える。  
> 特定カードや枚数条件でイベントを進める。  
> 終わった吹き出しは色を変え、次に見る場所を分かりやすくする。  
> 最終的に、カード同士の関係からSランク「飛鳥寺」を解放する。

これにより、Learning RPG は「歴史を読むゲーム」ではなく、**史実カードを使って時代を読み解くゲーム**になる。
