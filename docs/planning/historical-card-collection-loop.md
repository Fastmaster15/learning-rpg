# 史実カード収集ループ設計

## 1. 目的

Learning RPG では、史実をただ背景に置くのではなく、プレイヤーが町を歩いて発見し、**史実カード**として集めていく構造にする。

これにより、開発側の「史実ネタ収集」と、ゲーム内の「学びの収集体験」を同じ構造でつなげる。

## 2. 基本コンセプト

> 史実をカードとして集める。  
> ただし、クイズの正解報酬ではなく、町を歩き、見て、触れて、人と関わった結果として手に入る。

史実カードは、図鑑・コレクション・記録帳・学習ログを兼ねる。

プレイヤーは、時代の町で出会った建物、道具、服装、食、文化、事件、人物、技術などをカードとして集めていく。

## 3. 史実カードの役割

### 3.1 学びの成果

カードは、プレイヤーがその時代で実際に見つけたものの記録。

例：

- お寺の瓦
- 木簡
- 寺をつくる職人たち
- 飛鳥の市
- 仏像
- 宮に仕える人々

### 3.2 時代の解像度

カードが増えるほど、その時代の町の解像度が上がる。

単に「コンプリート率」を上げるだけではなく、カード同士がつながり、時代の構造が見えてくる。

例：

- 瓦 → 寺院建設 → 仏教文化 → 渡来系技術
- 木簡 → 文字 → 役人 → 物資管理
- 市 → 食材 → 供物 → 儀式

### 3.3 時代間のつながり

カードは、別の時代に行ったときにも意味を持つ。

例：

- 飛鳥で「瓦」を集める
- 奈良でより大きな寺院を見る
- 鎌倉で武士と寺社の関係を見る
- 室町で禅寺・茶室・庭へつながる

このとき、カードに「つながり」が追加される。

## 4. カード分類

史実カードは以下のカテゴリで管理する。

### 4.1 文化カード

信仰、儀式、芸能、文字文化、教育、茶、絵画など。

### 4.2 建物カード

寺、宮、屋敷、市、街道、港、城、町家、洋館など。

### 4.3 服装カード

冠、衣、武士の装束、町人の服、和装・洋装、履き物、髪型など。

### 4.4 食カード

米、雑穀、魚、塩、果実、木の実、蜜、茶、供物、保存食など。

### 4.5 道具・技術カード

木簡、筆、瓦、土器、農具、船、印刷、鉄道、新聞など。

### 4.6 事件・制度カード

政変、遷都、改革、幕府成立、税、身分制度、土地制度など。

### 4.7 戦争・争いカード

戦乱、内乱、外敵、火事、飢饉、疫病など。

### 4.8 人物カード

天皇、豪族、僧、武士、商人、職人、学者、芸能者など。

実名人物は慎重に扱い、最初は役割NPCや背景カードとして扱う。

## 5. カードの確度ラベル

ゲーム内カードにも、開発側では確度ラベルを持たせる。

### A：高確度

中核カードにできる。記録帳にも採用しやすい。

### B：中確度

背景・補助カードとして採用できる。

### C：低確度

演出候補。ゲーム内カードにはしない、または監修後に採用する。

### D：不採用候補

原則として使わない。

## 6. カード獲得の条件

カードは、ただ通り過ぎるだけでは手に入らない。

基本は以下のいずれかで獲得する。

- 近づいて見る
- 調べる
- NPCから話を聞く
- 短い手伝いをする
- その物が使われる場面を見る
- 関連する別カードを見つける

例：

### お寺の瓦

獲得条件：

- 職人の作業場で瓦を見る
- 職人に「屋根に使う」と聞く
- うーちゃんが近づいて注意されるイベントを見る

### 木簡

獲得条件：

- 役人が持っている木簡を見る
- 荷物についた木の札を調べる
- 「紙ではなく木に書く」ことに主人公が気づく

## 7. カード表示形式

### 表面

- イラスト
- カード名
- 時代
- カテゴリ
- 短い一言

例：

> お寺の瓦  
> 飛鳥 / 建物・技術  
> お寺の屋根に使われた、重くて丈夫な焼き物。

### 裏面

- もう少し詳しい説明
- 見つけた場所
- 関連カード
- 時代間リンク
- 史実メモ

## 8. 子ども向け説明と大人向けメモ

カードには2層の説明を持たせる。

### 子ども向け

短く、見たまま分かる説明。

例：

> お寺の屋根にのせる、かたい焼き物。たくさんの人が運んでいた。

### 大人向け / 詳細メモ

必要なら開ける補足。

例：

> 飛鳥時代の寺院建設では瓦が重要な建築素材として用いられた。文様や製作技術には、当時の仏教文化や大陸・朝鮮半島からの影響を考える手がかりがある。

## 9. カード同士のリンク

カードは単独で終わらせず、関連カードとつなぐ。

例：

- お寺の瓦 → 寺をつくる職人たち
- お寺の瓦 → 飛鳥の寺
- 木簡 → 役人
- 木簡 → 市の荷物
- 供物 → 儀式
- 供物 → 寺

リンクが増えるほど、時代の構造が見えてくる。

## 10. カード収集とゲーム進行

### 基本方針

カード収集は必須タスクにしすぎない。

ただし、主要カードを集めることで、町の理解が深まり、次の行動が自然に開く。

### 例

- 「お寺の瓦」を見る
- 「寺をつくる職人たち」を見る
- 「寺の人が来ない理由」へつながる
- 寺へ向かう意味が強くなる

## 11. Chapter 1 飛鳥の町の初期カード候補

### A候補

- お寺の瓦
- 寺をつくる職人たち
- 木簡
- 飛鳥の寺
- 仏像
- 宮に仕える人

### B候補

- 市の荷物
- 供物
- 素朴な器
- 木材を運ぶ人々
- 工人

### C候補

- 甘い香りの屋台
- 飛鳥のお菓子
- 子どもの遊び

C候補は、史実監修が済むまで中核カードにしない。

## 12. 開発用史実カードとの関係

ゲーム内カードは、開発用の史実ネタカードから派生する。

### 開発用カード

- 史実根拠
- 出典
- 確度
- 注意点
- 採用判断

### ゲーム内カード

- イラスト
- 子ども向け説明
- 発見場所
- 関連カード
- 時代間リンク

開発用カードを精査してから、ゲーム内カードに変換する。

## 13. 実装優先度

MVPでは以下を優先する。

1. カード一覧データ構造
2. 飛鳥のカード 5〜8枚
3. 発見済み / 未発見の状態管理
4. カード詳細画面
5. 関連カード表示

時代間リンクや大人向け詳細メモは、後続でよい。

## 14. 重要な設計思想

> プレイヤーは、歴史を読んで覚えるのではなく、町の中で見つけた史実をカードとして集めていく。

カード収集は、単なるコレクションではない。

時代の町を歩き、物を見て、人に出会い、文化のつながりを少しずつ理解するための仕組みである。
