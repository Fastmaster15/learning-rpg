# 史実カード 時代タグ・カードパック設計

## 1. 目的

Learning RPG の史実カードには、時代タグを付ける。

また、カードを単体で集めるだけでなく、時代別・テーマ別のカードパックとして管理する。

これにより、プレイヤーは以下を直感的に理解できる。

- いまどの時代のカードを集めているか
- その時代にカードが何枚あるか
- どのカテゴリが埋まっているか
- どのテーマがまだ未発見か
- 時代を超えてつながるカードがどれか

## 2. 基本方針

> カードは、時代タグで整理する。  
> カードパックは、収集目標として見せる。  
> 総数は最初に分かる。  
> ただし、未発見カードの詳細は伏せる。

カードパックは、ランダムガチャではなく、図鑑・アルバム・章ごとの収集ボードとして扱う。

## 3. 時代タグ

### 3.1 基本タグ

各カードには、必ず主時代タグを付ける。

例：

- 飛鳥
- 奈良
- 平安
- 鎌倉
- 室町
- 戦国
- 江戸
- 明治
- 大正
- 昭和
- 平成
- 令和

### 3.2 複数時代タグ

一部のカードは、複数時代にまたがる。

例：

- 仏教文化：飛鳥 / 奈良 / 鎌倉 / 室町
- 文字と記録：飛鳥 / 奈良 / 平安 / 江戸 / 明治
- 寺院建築：飛鳥 / 奈良 / 鎌倉 / 室町

この場合、カードには以下を持たせる。

- 初出・主時代
- 関連時代
- 時代横断タグ

### 3.3 タグ構造案

```ts
{
  id: "asuka-roof-tile",
  name: "お寺の瓦",
  primaryEra: "飛鳥",
  relatedEras: ["奈良", "鎌倉", "室町"],
  categories: ["道具", "技術", "建物"],
  themes: ["寺院建築", "仏教文化", "外来技術"],
  rarity: "A",
  certainty: "A",
  quantityType: "stackable"
}
```

## 4. カードパックの種類

### 4.1 時代パック

時代ごとの基本パック。

例：

- 飛鳥パック
- 奈良パック
- 平安パック
- 鎌倉パック
- 室町パック
- 江戸パック
- 明治パック

### 4.2 テーマパック

時代を横断するテーマ別パック。

例：

- 仏教文化パック
- 文字と記録パック
- 建物と技術パック
- 食と暮らしパック
- 人物と政治パック
- 服装と身分パック
- 戦争と争いパック

### 4.3 章パック

ストーリー進行に対応するパック。

例：

- Chapter 1：飛鳥の町パック
- Chapter 2：奈良の都パック
- Chapter 3：平安の都パック

章パックは、実際のプレイ導線とカード収集目標を一致させる。

## 5. 総数表示

### 5.1 基本方針

カード総数は、最初に分かる方がよい。

理由：

- 収集目標が明確になる
- 進捗が見える
- もう一度探索する理由になる
- 親子で「あと何枚？」と話しやすい

### 5.2 表示例

#### 時代パック

> 飛鳥パック 8 / 32

#### カテゴリ別

> 人物 2 / 6  
> 建物 1 / 4  
> 道具・技術 3 / 8  
> 食 1 / 5  
> 服装 0 / 4  
> 事件・制度 1 / 5

#### レアリティ別

> S 0 / 4  
> A 4 / 10  
> B 4 / 14  
> C 0 / 4

## 6. 未発見カードの見せ方

### 6.1 総数は見せるが、詳細は伏せる

未発見カードは、名前をすべて見せない。

表示案：

- シルエット
- 「未発見カード」
- カテゴリだけ表示
- ヒントだけ表示

例：

> 未発見 Sカード 1 / 建物  
> ヒント：町の奥に見える、大きなお寺に関係する。

### 6.2 ネタバレを避けるカード

Sランクカードや事件カードは、最初から名前を出しすぎない。

例：

- 「飛鳥寺」と最初から出すのではなく、序盤は「大きなお寺のカード」
- 関連カードが集まると正式名称が開く

## 7. パック完成報酬

パック完成報酬は、史実を崩さない範囲で設定する。

### 報酬例

- 特別絵柄カード
- 時代横断リンク解放
- カード裏面の詳細メモ解放
- うーちゃんのコメント解放
- 章のまとめ絵巻解放
- 関連時代へのヒント解放

### 避けること

- 史実と無関係な強化アイテム
- 国宝級カードをガチャ報酬のように扱う
- ランダム性だけで入手させる

## 8. 絵柄違いと時代タグ

絵柄違いカードにも時代タグを付ける。

ただし、絵柄違いは単なるレア演出ではなく、文脈違いとして扱う。

### 例：木簡

- 飛鳥 / 市の荷物についた木簡
- 飛鳥 / 役人が持つ木簡
- 奈良 / 役所で使われる文書
- 江戸 / 瓦版との比較カード

### 例：瓦

- 飛鳥 / 職人の作業場で見る瓦
- 飛鳥 / 寺の屋根に使われる瓦
- 奈良 / 大寺院の瓦
- 室町 / 禅寺の屋根との比較カード

## 9. カードパックと繰り返しプレイ

### 9.1 一度クリアしても戻る理由

カード総数と未発見枠が見えることで、プレイヤーはもう一度町へ戻る理由ができる。

例：

- 飛鳥パック 24 / 32
- 食カード 2 / 5
- Sカード 1 / 4
- 絵柄違い 3 / 8

### 9.2 ストーリー再訪

クリア後に町へ戻ると、未発見カードや別絵柄を探せる。

ただし、ランダム周回ではなく、以下のような再訪理由を作る。

- まだ行っていない場所
- 別の時間帯
- 別のNPC会話
- 既に持っているカードによる新しい気づき
- 時代横断カードによる再解釈

## 10. Chapter 1 飛鳥パック初期案

### 仮総数

MVP初期案では、飛鳥パックは **24〜32枚程度** を目安にする。

内訳案：

- S：3〜4枚
- A：8〜10枚
- B：10〜14枚
- C：3〜4枚

### 初期表示例

> 飛鳥パック 0 / 28

カテゴリ：

- 人物 0 / 6
- 建物 0 / 4
- 道具・技術 0 / 7
- 食・供物 0 / 4
- 服装 0 / 3
- 事件・制度 0 / 4

### 最初に見えるカード枠

- 市の荷物
- 供物
- お寺の瓦
- 寺をつくる職人たち
- 木簡
- 役人
- 僧

## 12. データ設計メモ

### 12.1 Card

```ts
type Card = {
  id: string;
  name: string;
  primaryEra: Era;
  relatedEras: Era[];
  categories: CardCategory[];
  themes: string[];
  rarity: "S" | "A" | "B" | "C" | "D";
  certainty: "A" | "B" | "C" | "D";
  quantityType: "unique" | "stackable" | "variant";
  maxCopies?: number;
  packIds: string[];
  hiddenNameUntilDiscovered?: boolean;
};
```

### 12.2 CardPack

```ts
type CardPack = {
  id: string;
  name: string;
  type: "era" | "theme" | "chapter";
  era?: Era;
  theme?: string;
  cardIds: string[];
  visibleTotal: boolean;
  hideUndiscoveredNames: boolean;
};
```

## 13. 失敗条件

以下は避ける。

- カードパックがガチャのように見える
- 総数が見えず、収集目標が分からない
- 逆に未発見カード名を出しすぎてネタバレになる
- 時代タグが雑で、時代横断リンクが誤解を生む
- 絵柄違いが単なる水増しになる
- Cランク保留カードが目立ちすぎる

## 14. 重要な設計思想

> 時代タグは、カードを歴史の流れに置くための軸。  
> カードパックは、プレイヤーが時代の解像度を上げていくための地図。

総数が見えることで集めたくなる。  
未発見が残ることで探したくなる。  
時代タグがあることで、別の時代へ行ったときにまた意味が生まれる。
