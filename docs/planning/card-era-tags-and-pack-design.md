# 史実カード 時代タグ・カードパック設計

## 1. 目的

Learning RPG の史実カードには、時代タグを付ける。

また、カードを単体で集めるだけでなく、時代別・テーマ別のカードパックとして管理する。

これにより、プレイヤーは以下を直感的に理解できる。

- いまどの時代のカードを集めているか
- その時代にカードが何枚あるか
- どのカテゴリが埋まっているか
- どのテーマがまだ未発見か
- 時代を超えてつながるカードがどれか

ただし、時代を超えたつながりは最初からすべて見せない。

プレイヤーが複数の時代を旅し、関連カードを見つけたときに、カードへ新しい時代タグや時代間リンクが追加されていく形にする。

## 2. 基本方針

> カードは、時代タグで整理する。  
> カードパックは、収集目標として見せる。  
> 総数は最初に分かる。  
> ただし、未発見カードの詳細は伏せる。  
> 時代横断タグは、時代を重ねることで後から開く。

カードパックは、ランダムガチャではなく、図鑑・アルバム・章ごとの収集ボードとして扱う。

## 3. 時代タグ

### 3.1 基本タグ

各カードには、必ず主時代タグを付ける。

例：

- 飛鳥
- 奈良
- 平安
- 鎌倉
- 室町
- 戦国
- 江戸
- 明治
- 大正
- 昭和
- 平成
- 令和

### 3.2 関連時代タグは後から追加される

一部のカードは、複数時代にまたがる意味を持つ。

例：

- 仏教文化：飛鳥 / 奈良 / 鎌倉 / 室町
- 文字と記録：飛鳥 / 奈良 / 平安 / 江戸 / 明治
- 寺院建築：飛鳥 / 奈良 / 鎌倉 / 室町

ただし、プレイヤーには最初から全時代タグを表示しない。

最初は、発見した時代だけが見える。

別の時代で関連カードを見つけたときに、カードへ新しい関連時代タグが追加される。

### 例：お寺の瓦

初回、飛鳥で発見した時点：

> お寺の瓦  
> 時代タグ：飛鳥

奈良で大寺院の瓦や寺院建築に触れた後：

> お寺の瓦  
> 時代タグ：飛鳥  
> 追加リンク：奈良

室町で禅寺や寺院建築に触れた後：

> お寺の瓦  
> 時代タグ：飛鳥  
> 追加リンク：奈良 / 室町

### 狙い

- 最初からネタバレしない
- 時代を旅するほど、過去カードの意味が増える
- 前に集めたカードを見返す理由ができる
- 「同じテーマが時代ごとに形を変える」感覚が出る
- プレイヤー自身がつながりを発見したように感じられる

### 3.3 内部データと表示データを分ける

開発側の内部データでは、関連時代を持ってよい。

ただし、ゲーム画面では未解放の関連時代は伏せる。

```ts
{
  id: "asuka-roof-tile",
  name: "お寺の瓦",
  primaryEra: "飛鳥",
  hiddenRelatedEras: ["奈良", "鎌倉", "室町"],
  unlockedEraLinks: [],
  categories: ["道具", "技術", "建物"],
  themes: ["寺院建築", "仏教文化", "外来技術"],
  rarity: "A",
  certainty: "A",
  quantityType: "stackable"
}
```

プレイヤーが後の時代で関連カードを発見すると、`unlockedEraLinks` に追加される。

例：

```ts
unlockedEraLinks: ["奈良"]
```

## 4. カードパックの種類

### 4.1 時代パック

時代ごとの基本パック。

例：

- 飛鳥パック
- 奈良パック
- 平安パック
- 鎌倉パック
- 室町パック
- 江戸パック
- 明治パック

### 4.2 テーマパック

時代を横断するテーマ別パック。

例：

- 仏教文化パック
- 文字と記録パック
- 建物と技術パック
- 食と暮らしパック
- 人物と政治パック
- 服装と身分パック
- 戦争と争いパック

テーマパックは、最初から全カード数を見せすぎない。

複数時代を進むことで、テーマパックの見えている範囲が広がる。

### 4.3 章パック

ストーリー進行に対応するパック。

例：

- Chapter 1：飛鳥の町パック
- Chapter 2：奈良の都パック
- Chapter 3：平安の都パック

章パックは、実際のプレイ導線とカード収集目標を一致させる。

## 5. 総数表示

### 5.1 基本方針

カード総数は、最初に分かる方がよい。

理由：

- 収集目標が明確になる
- 進捗が見える
- もう一度探索する理由になる
- 親子で「あと何枚？」と話しやすい

ただし、総数表示には段階を持たせる。

- 時代パック：最初から総数を見せる
- 章パック：最初から総数を見せる
- テーマパック：未到達時代のカード数は伏せてもよい
- 時代横断リンク数：最初から総数を見せない

### 5.2 表示例

#### 時代パック

> 飛鳥パック 8 / 32

#### カテゴリ別

> 人物 2 / 6  
> 建物 1 / 4  
> 道具・技術 3 / 8  
> 食 1 / 5  
> 服装 0 / 4  
> 事件・制度 1 / 5

#### レアリティ別

> S 0 / 4  
> A 4 / 10  
> B 4 / 14  
> C 0 / 4

#### 時代横断リンク

> 時代リンク：未発見

または、関連時代を一度見つけた後：

> 時代リンク：奈良 1件

## 6. 未発見カードの見せ方

### 6.1 総数は見せるが、詳細は伏せる

未発見カードは、名前をすべて見せない。

表示案：

- シルエット
- 「未発見カード」
- カテゴリだけ表示
- ヒントだけ表示

例：

> 未発見 Sカード 1 / 建物  
> ヒント：町の奥に見える、大きなお寺に関係する。

### 6.2 ネタバレを避けるカード

Sランクカードや事件カードは、最初から名前を出しすぎない。

例：

- 「飛鳥寺」と最初から出すのではなく、序盤は「大きなお寺のカード」
- 関連カードが集まると正式名称が開く

### 6.3 未解放の時代リンクも伏せる

時代横断カードについても、未到達時代のリンクは最初から表示しない。

悪い例：

> お寺の瓦：飛鳥 / 奈良 / 鎌倉 / 室町

良い例：

> お寺の瓦：飛鳥

後に解放：

> お寺の瓦：飛鳥  
> 新しい時代リンク：奈良

## 7. パック完成報酬

パック完成報酬は、史実を崩さない範囲で設定する。

### 報酬例

- 特別絵柄カード
- 時代横断リンク解放
- カード裏面の詳細メモ解放
- うーちゃんのコメント解放
- 章のまとめ絵巻解放
- 関連時代へのヒント解放

### 避けること

- 史実と無関係な強化アイテム
- 国宝級カードをガチャ報酬のように扱う
- ランダム性だけで入手させる

## 8. 絵柄違いと時代タグ

絵柄違いカードにも時代タグを付ける。

ただし、絵柄違いは単なるレア演出ではなく、文脈違いとして扱う。

### 例：木簡

初期表示：

- 飛鳥 / 市の荷物についた木簡
- 飛鳥 / 役人が持つ木簡

後の時代で解放：

- 奈良 / 役所で使われる文書
- 江戸 / 瓦版との比較カード

### 例：瓦

初期表示：

- 飛鳥 / 職人の作業場で見る瓦
- 飛鳥 / 寺の屋根に使われる瓦

後の時代で解放：

- 奈良 / 大寺院の瓦
- 室町 / 禅寺の屋根との比較カード

## 9. カードパックと繰り返しプレイ

### 9.1 一度クリアしても戻る理由

カード総数と未発見枠が見えることで、プレイヤーはもう一度町へ戻る理由ができる。

例：

- 飛鳥パック 24 / 32
- 食カード 2 / 5
- Sカード 1 / 4
- 絵柄違い 3 / 8

### 9.2 ストーリー再訪

クリア後に町へ戻ると、未発見カードや別絵柄を探せる。

ただし、ランダム周回ではなく、以下のような再訪理由を作る。

- まだ行っていない場所
- 別の時間帯
- 別のNPC会話
- 既に持っているカードによる新しい気づき
- 後の時代で解放された時代横断リンクによる再解釈

### 9.3 後の時代に行ったあと、過去カードが更新される

奈良・平安・室町などへ進んだ後、飛鳥で集めたカードに新しいリンクが追加される。

例：

- 飛鳥で「お寺の瓦」を集める
- 奈良で大寺院の瓦を見る
- 「お寺の瓦」に奈良リンクが追加される
- 飛鳥パックを見返すと、カードに新しい時代タグが増えている

この仕組みにより、過去のカードが死なない。

## 10. Chapter 1 飛鳥パック初期案

### 仮総数

MVP初期案では、飛鳥パックは **24〜32枚程度** を目安にする。

内訳案：

- S：3〜4枚
- A：8〜10枚
- B：10〜14枚
- C：3〜4枚

### 初期表示例

> 飛鳥パック 0 / 28

カテゴリ：

- 人物 0 / 6
- 建物 0 / 4
- 道具・技術 0 / 7
- 食・供物 0 / 4
- 服装 0 / 3
- 事件・制度 0 / 4

### 最初に見えるカード枠

- 市の荷物
- 供物
- お寺の瓦
- 寺をつくる職人たち
- 木簡
- 役人
- 僧
- 未発見Sカード

時代横断リンクは、Chapter 1開始時点では原則未表示。

## 12. データ設計メモ

### 12.1 Card

```ts
type Card = {
  id: string;
  name: string;
  primaryEra: Era;
  hiddenRelatedEras: Era[];
  categories: CardCategory[];
  themes: string[];
```

## 13. 失敗条件

以下は避ける。

- カードパックがガチャのように見える
- 総数が見えず、収集目標が分からない
- 逆に未発見カード名を出しすぎてネタバレになる
- 時代タグが雑で、時代横断リンクが誤解を生む
- 最初から未来時代のリンクを全部見せてしまい、発見感がなくなる
- 絵柄違いが単なる水増しになる
- Cランク保留カードが目立ちすぎる

## 14. 重要な設計思想

> 時代タグは、カードを歴史の流れに置くための軸。  
> カードパックは、プレイヤーが時代の解像度を上げていくための地図。  
> 時代横断リンクは、旅を重ねることで増えていく発見である。

総数が見えることで集めたくなる。  
未発見が残ることで探したくなる。  
時代タグが後から増えることで、前に集めたカードを見返したくなる。
