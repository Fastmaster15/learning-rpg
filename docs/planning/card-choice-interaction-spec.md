# カード3択インタラクション仕様

## 1. 目的

Learning RPG では、イベントやNPC会話の進行時に、プレイヤーが手持ちの史実カードから1枚を選んで提示する。

ただし、毎回すべてのカードから選ばせると重くなるため、基本は **3択** にする。

この仕組みにより、カード収集とストーリー進行を結びつける。

## 2. 基本コンセプト

> 「どれを選ぶ？」  
> 手持ちカードから3枚が提示される。  
> 選んだカードによって、NPCやイベントの反応が変わる。

カード選択はクイズではない。

プレイヤーが「この場面ならこのカードが関係ありそう」と考える、小さな推理・観察・関連づけの遊びにする。

## 3. 3択にする理由

### 3.1 選択負荷を下げる

全カードから選ばせると、子どもには難しく、テンポも悪くなる。

3択なら、直感的に選びやすい。

### 3.2 カードを使っている感が出る

自動参照だけでは、カードを持っている意味が弱い。

3択で選ばせることで、プレイヤーがカードを使ってイベントを進めた感覚が出る。

### 3.3 誤答ではなく反応差分にできる

3択のすべてに何らかの反応を用意することで、単純な正解・不正解にしない。

## 4. カード候補の出し方

### 4.1 手持ちカードから出す

提示されるカードは、原則としてプレイヤーが既に持っているカードのみ。

未所持カードは選択肢に出さない。

### 4.2 候補は状況に応じて絞る

イベント側で、関連度の高いカードを候補として選ぶ。

例：職人イベントでは以下が出やすい。

- お寺の瓦
- 木材を運ぶ人々
- 寺をつくる職人たち

市の荷物イベントでは以下が出やすい。

- 市の荷物
- 供物
- 木簡

### 4.3 3択の構成

基本構成は以下。

1. **本命カード**：イベント進行に強く関係するカード
2. **関連カード**：会話や観察は増えるが、進行の本筋ではないカード
3. **少しズレたカード**：完全に無関係ではないが、今の場面では弱いカード

この構成により、選択に意味を持たせる。

## 5. 反応タイプ

カード選択後の反応は、主に3種類に分ける。

## 5.1 強反応

イベント進行に大きく関係するカードを選んだ場合。

### 効果

- NPC会話が進む
- 新しい情報が出る
- 関連カードがつながる
- Sカードの解放条件が進む
- 場所や次の行動が開く

### 例

職人に「お寺の瓦」を見せる。

職人：
> 「お、よく見ていたね。これはお寺の屋根に使うんだよ。」

表示：
> 「お寺の瓦」と「寺をつくる職人たち」がつながった！

## 5.2 中反応

関連はあるが、本筋ではないカードを選んだ場合。

### 効果

- 追加会話が少し出る
- カード理解度が少し上がる
- ただしイベントの本筋は大きく進まない

### 例

職人に「木材を運ぶ人々」を見せる。

職人：
> 「そうそう、木も大事だよ。瓦だけじゃお寺は建たないからね。」

表示：
> 建物づくりの手がかりが少し増えた。

## 5.3 弱反応

今の場面ではあまり関係が強くないカードを選んだ場合。

### 効果

- 軽いリアクションのみ
- うーちゃんが別のカードを促す
- 不正解扱いではなく、考える余地を残す

### 例

職人に「供物」を見せる。

職人：
> 「それはお寺の人に聞いた方がいいかもしれないね。」

うーちゃん：
> 「これは別の人に見せるとよさそう！」

## 6. UI仕様

## 6.1 会話中の表示

NPC会話中、必要な場面でカード選択を出す。

```text
職人：「こらこら、近づきすぎると危ないよ。」

うーちゃん：「なにを見せて聞いてみる？」

どれを選ぶ？

[お寺の瓦]
[木材を運ぶ人々]
[供物]
```

### UI方針

- 画面下部に3枚のカードを横並び、または縦並びで出す
- カード名、カテゴリ、短いタグを表示する
- レアリティ演出は控えめにする
- 子どもでも押しやすいサイズにする

## 6.2 場所調査時の表示

```text
寺の入口に来た。
屋根の上で、何かがきらっと光った。

どのカードを使って見てみる？

[お寺の瓦]
[供物]
[木簡]
```

## 6.3 重要イベント時の表示

章終盤では、単発3択ではなく、複数回の3択で関係を作る。

```text
寺の人が市へ来なかった理由を考えよう。

まず、荷物に関係ありそうなカードは？

[市の荷物]
[お寺の瓦]
[僧]
```

次の選択：

```text
その荷物は、どこへ向かうものだった？

[供物]
[木材を運ぶ人々]
[役人]
```

このように、3択を連続させることでカード接続パズルにする。

## 7. 候補カード選定ルール

### 7.1 基本ルール

各イベントには、カード候補テーブルを持たせる。

```ts
const eventCardChoices = {
  eventId: "asuka-craftsperson-worksite",
  choices: [
    {
      cardId: "asuka-roof-tile",
      reactionType: "strong",
      unlocks: ["link-roof-tile-craftsperson", "asuka-temple-progress"]
    },
    {
      cardId: "asuka-timber-carriers",
      reactionType: "medium",
      unlocks: ["building-material-note"]
    },
    {
      cardId: "asuka-offering",
      reactionType: "weak",
      redirectHint: "temple-monk"
    }
  ]
}
```

### 7.2 手持ちにないカードがある場合

候補カードをプレイヤーが持っていない場合は、代替候補を出す。

ただし、本命カードを持っていない場合は、イベントが完全に詰まらないようにする。

### 7.3 詰み防止

本命カードを持っていない場合でも、以下のいずれかで進める。

- 関連カードで中反応を出し、別の場所へ誘導する
- うーちゃんが「先にあっちを見てみよう」と促す
- イベントは保留になり、後で再挑戦できる

## 8. 3択のパターン

## 8.1 本命あり型

最も基本的な形。

- 本命カード
- 関連カード
- 弱関連カード

### 例

職人イベント：

- お寺の瓦：強反応
- 木材を運ぶ人々：中反応
- 供物：弱反応

## 8.2 どれも有効型

3枚すべてが異なる方向に進む。

### 例

寺へ向かう前の分岐：

- 木簡：役人ルート
- お寺の瓦：職人ルート
- 供物：僧ルート

この場合、どれを選んでも進むが、先に見るイベントが変わる。

## 8.3 ヒント型

まだ本命カードを持っていない場合。

- 関連カード
- 関連カード
- 少しズレたカード

選択後、うーちゃんやNPCが本命カードのありそうな場所を示す。

## 8.4 章終盤パズル型

3択を複数回行い、カードの関係を段階的に作る。

これはChapter 1終盤のカード接続パズルで使う。

## 9. Chapter 1の具体例

## 9.1 市の一角：寺へ届ける荷物

### 状況

店の人が、寺の人が来ないことを気にしている。

### 3択

```text
どのカードを見せて聞いてみる？

[市の荷物]
[供物]
[お寺の瓦]
```

### 反応

#### 市の荷物：強反応

店の人：
> 「そう、それはお寺へ向かう荷なんだ。」

効果：

- 市の荷物と寺がつながる
- 寺へ向かう導線が強まる

#### 供物：中反応

店の人：
> 「うん、これは大事な場で使うものなんだ。」

効果：

- 供物の理解度が上がる
- ただし、寺へ向かう理由はまだ弱い

#### お寺の瓦：弱反応

店の人：
> 「それは職人さんたちの方が詳しいかもね。」

効果：

- 職人作業場へのヒント

## 9.2 職人の作業場

### 3択

```text
職人さんに、どのカードを見せる？

[お寺の瓦]
[木材を運ぶ人々]
[供物]
```

### 反応

#### お寺の瓦：強反応

職人：
> 「これはお寺の屋根に使うんだ。たくさん必要なんだよ。」

効果：

- お寺の瓦と寺をつくる職人たちがつながる
- 飛鳥寺Sカードの輪郭が少し明るくなる

#### 木材を運ぶ人々：中反応

職人：
> 「木も大事だよ。柱や足場に使うからね。」

効果：

- 建物づくりラインが少し進む

#### 供物：弱反応

職人：
> 「それはお坊さんの方が詳しいかな。」

効果：

- 僧ルートへのヒント

## 9.3 役人・木簡イベント

### 3択

```text
役人さんに、どのカードを見せる？

[木簡]
[市の荷物]
[お寺の瓦]
```

### 反応

#### 木簡：強反応

役人：
> 「これは荷の行き先や数をたしかめるために使うんだ。」

効果：

- 木簡と役人がつながる
- 記録・管理ラインが開く

#### 市の荷物：中反応

役人：
> 「その荷も、どこへ行くか確かめないとね。」

効果：

- 市の荷物と記録の関係に近づく

#### お寺の瓦：弱反応

役人：
> 「それは作業場の人に聞くといい。」

効果：

- 職人ルートへのヒント

## 9.4 寺の入口：章終盤パズル

### 段階1

```text
寺へ届くはずだったものは？

[供物]
[お寺の瓦]
[木簡]
```

### 段階2

```text
それを受け取るはずだった人たちは？

[僧]
[役人]
[職人]
```

### 段階3

```text
寺がばたばたしていた理由に関係ありそうなのは？

[寺をつくる職人たち]
[市の荷物]
[宮に仕える人]
```

### 結果

正しいラインが成立すると、次の理解が開く。

> 寺では、儀式の準備と建設作業が重なっていた。  
> 市から届く荷、供物、職人たちの作業、僧たちの準備が一つの場所に集まっていた。

Sランク「飛鳥寺」カードの解放へ進む。

## 10. 反応差分の設計原則

### 10.1 すべての選択に意味を持たせる

弱反応でも、完全なハズレにはしない。

- 別ルートのヒント
- 別NPCへの誘導
- そのカードの使いどころの示唆
- うーちゃんの軽いリアクション

を入れる。

### 10.2 正解・不正解ではなく、関係の強弱にする

カード選択は歴史クイズではない。

反応は以下のように扱う。

- 強い関係
- 近い関係
- 今は遠い関係

### 10.3 史実的に誤った接続は成立させない

弱反応であっても、誤った歴史理解を与えない。

「これは違う」と否定するより、

> 「それは別の場面で使えそう」

と誘導する。

## 11. 実装優先度

### MVPで実装

- イベントごとの3択カード提示
- 強反応 / 中反応 / 弱反応
- 手持ちカードから候補表示
- 選択結果による会話差分
- Sカード進行度の更新

### 後続で実装

- ルート分岐型3択
- 複数段階カード接続パズル
- 選択履歴によるNPC記憶
- 対戦・協力モードへの転用

## 12. 失敗条件

以下は避ける。

- 毎回全カードから選ばせてテンポが悪くなる
- 3択が単なる正解当てになる
- 弱反応が完全なハズレ扱いになる
- 持っていないカードが必要で詰む
- どのカードを選んでも同じ反応になる
- 史実的に誤った関係が成立する
- UIが重く、会話テンポを壊す

## 13. 重要な設計思想

> カード選択は、歴史クイズではなく、関係の強さを探す遊びである。

3択にすることで、子どもでも選びやすくする。

反応差分を入れることで、大人も「このカードならどう反応するか」を楽しめる。

これにより、史実カードは収集物ではなく、会話とイベントを動かす道具になる。
