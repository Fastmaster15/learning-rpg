# Learning RPG 史実ネタ収集・展開設計

## 1. 目的

Learning RPG は、先に物語やクエストを作ってから歴史要素を当てはめるのではなく、まず史実的に確からしいネタを集め、その中からゲーム体験へ展開する。

このドキュメントは、史実ネタの収集方法、分類、確度管理、ゲーム化の流れを定義する。

## 2. 基本方針

### 2.1 史実ファースト

この企画では、史実・文化・生活・建物・服装・食・事件・戦争などを先に収集する。

そこから、以下へ展開する。

- 時代絵巻
- 町の構造
- NPC
- クエスト
- 記録帳カード
- アイテム
- 会話
- イベント
- 時代間のつながり

### 2.2 ゲーム化は二段階で行う

1. 史実ネタを集める
2. そのネタを、子どもが体験できる町歩き・会話・発見・記録に変換する

いきなり「面白いイベント」を作らない。

まず、史実の中にある面白さを見つける。

## 4. 史実ネタカードの形式

各ネタは、以下の形式で管理する。

```md
## ネタ名

### 時代

### カテゴリ

文化 / 建物 / 服装 / 食 / 道具 / 事件 / 戦争 / 人物 など

### 概要

### 史実根拠

- 参考資料
- 出典
- 確認メモ

### 確度

A / B / C / D

### 子どもに伝わる見せ方

### RPG化アイデア

### 記録帳カード案

### 注意点
```

## 5. 確度ラベル

### A：高確度

史料・考古資料・博物館資料・自治体資料などで確認しやすいもの。

ゲームの中核イベントや記録帳カードにしてよい。

### B：中確度

時代背景として十分あり得るが、表現には注意が必要なもの。

背景や補助イベントとして使う。

### C：低確度

魅力はあるが、裏取りが弱いもの。

中核イベントにはしない。使う場合は曖昧な背景表現に留める。

### D：不採用候補

時代錯誤、または誤解を招く可能性が高いもの。

原則使わない。

## 6. 史実ネタからゲーム体験への変換ルール

### 6.1 まず「見えるもの」に変換する

抽象概念をいきなり説明しない。

例：

- 仏教伝来 → 寺、瓦、仏像、僧、儀式、職人
- 文字文化 → 木簡、筆、役人、届け物
- 交易・外来文化 → 見慣れない道具、服装、素材、言葉づかい

### 6.2 次に「触れるもの」に変換する

プレイヤーが調べたり、近づいたり、少し手伝える形にする。

例：

- 瓦を見る
- 木簡を拾う
- 職人に話しかける
- 市の荷物を見せてもらう
- 儀式の準備を遠くから見る

### 6.3 最後に「記録」に変換する

体験したあと、短い記録帳カードとして残す。

記録帳は、ゲーム中の体験をあとから意味づけする場所。

## 7. Chapter 1 飛鳥の町で優先して集めるネタ

### 高優先

- 飛鳥寺
- 寺院建設
- 瓦
- 木簡
- 仏教文化
- 豪族
- 渡来系技術者・工人
- 市や物資のやりとり
- 食材・供物

### 中優先

- 服装
- 住居
- 土器や器
- 道具
- 子どもの生活
- 町の音や作業風景

### 要注意

- 甘い屋台
- 具体的な菓子
- 子どもの遊び
- 庶民の食事の細部
- 会話の言葉づかい

## 8. Chapter 1 の再設計方針

今後の Chapter 1 は、以下の順番で再設計する。

1. 飛鳥時代の史実ネタを収集する
2. 確度A/Bのネタだけで時代絵巻の骨格を作る
3. 確度Cのネタは演出候補として保留する
4. クエストは史実的に強いネタから作る
5. 記録帳カードは確度A/Bを優先する

## 9. 重要な設計思想

> 歴史をゲームにするのではなく、史実の中にある手触りをゲーム体験に変換する。

このRPGの価値は、史実をゆがめて面白くすることではない。

史実の中にある文化・暮らし・技術・人間関係・変化そのものを、子どもが歩いて感じられる形にすることにある。
