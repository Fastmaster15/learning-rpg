# Learning RPG ゲーム化補完仕様

## 1. 目的

このドキュメントは、Learning RPG を「面白いカード学習企画」から、実際に遊べるゲームへ落とし込むための補完仕様である。

現時点で強い設計は以下である。

- 史実性を最優先する
- ストーリー仕立てで時代の町を歩く
- 史実カードを集める
- カード同士をつなげる
- 時代をまたぐことでカードの意味が増える
- Sランクカードや絵柄違いで収集欲を作る

一方で、ゲーム化には以下が必要になる。

- プレイヤーの選択
- 失敗・取り逃し・再挑戦
- カード効果
- カード接続パズル
- 章ごとの勝利条件
- カード帳UI
- 最初の5分の体験設計

## 2. ゲームの基本定義

### 2.1 一文定義

> ストーリーで時代に入り込み、史実カードを集め、つなげ、使うことで、時代の出来事を読み解くカードRPG。

### 2.2 重要な設計思想

> カードを集めるゲームではなく、カードで時代を読み解くゲームにする。

カード収集は目的ではなく、時代理解を進めるための手段である。

## 3. コアゲームループ

### 3.1 基本ループ

1. 時代の町に入る
2. 町で出来事に出会う
3. 人・物・建物・食・服装・制度を観察する
4. 史実カードを獲得する
5. カード同士がつながる
6. カードを手がかりに町の出来事を読み解く
7. 章のSランクカードや次時代へのリンクが開く
8. 未発見カードや絵柄違いを探すために再訪する

### 3.2 目指す体験

- 読んで覚えるのではなく、歩いて気づく
- クイズではなく、カード関係で理解する
- レアカードはごほうびではなく、時代の核心として扱う
- 通常カードも、Sカードを理解するための重要な手がかりにする

## 4. プレイヤーの選択

### 4.1 必要性

プレイヤーがただ町を歩いて自動的にカードを得るだけでは、ゲームではなくインタラクティブ図鑑に近くなる。

ゲームとして成立させるため、プレイヤーに小さな選択を与える。

### 4.2 選択の種類

- どのNPCに先に話すか
- どの場所を先に調べるか
- どのカードを手がかりとして使うか
- 寺へ直行するか、市や作業場をもう少し見るか
- どのテーマパックを優先して埋めるか
- 限られた行動回数の中で、どのカードを探すか

### 4.3 MVPでの方針

Chapter 1 MVPでは、選択肢を複雑にしすぎない。

最低限、以下を入れる。

- 市の一角を調べる
- 寺へ向かう
- 道中の作業場を調べる
- 役人に話す
- 寺の入口へ進む
- カード接続パズルに挑む

## 5. 失敗・取り逃し・再挑戦

### 5.1 基本方針

歴史学習ゲームなので、理不尽な失敗やゲームオーバーは不要。

ただし、取り逃しや未完成状態は必要。

### 5.2 取り逃しの種類

- その場では気づけないカードがある
- 別カードを持っていると見えるカードがある
- 時間帯やルートで出会えるNPCが変わる
- 1周目では未完成のカードがある
- 関連カード不足でSランクカードがシルエットのまま残る

### 5.3 再挑戦の動機

- 未発見カード枠
- 絵柄違い
- 発見場所違い
- カード完成度
- Sカードの未解放
- 時代横断リンクの追加

### 5.4 失敗の扱い

失敗は「不正解」ではなく「まだ材料が足りない」と扱う。

例：

> まだこのカードだけでは、寺で何が起きているか分からない。  
> ほかの手がかりを探してみよう。

## 6. カード効果

### 6.1 基本方針

カード効果は、攻撃力や魔法効果ではない。

カード効果は、プレイヤーの観察・会話・接続を広げる。

### 6.2 効果分類

#### 観察効果

持っていると、町で見えるポイントが増える。

例：

- 木簡カードを持っていると、市の荷物についた札に気づける
- お寺の瓦カードを持っていると、寺の屋根や作業場に注目できる

#### 会話効果

持っていると、NPCとの会話が増える。

例：

- お寺の瓦カードを持っていると、職人が瓦の使い道を話してくれる
- 木簡カードを持っていると、役人が記録や荷物管理について少し話す

#### 接続効果

関連カードがつながり、町の出来事を読み解ける。

例：

- 供物 + 僧 + 飛鳥寺 = 儀式準備
- 木簡 + 役人 + 市の荷物 = 物資管理
- 瓦 + 職人 + 工人 = 寺院建設の技術

### 6.3 実装上の注意

- 史実的に確認された関係だけを接続する
- プレイヤーが自由に誤った歴史関係を作れる設計にしない
- カード効果文は短くする
- 効果は必ずゲーム内の行動に反映する

## 7. カード接続パズル

### 7.1 目的

対戦モードの前段階として、カード接続パズルを実装する。

これは暗記クイズではなく、集めた史実カードを使って町の出来事を読み解く仕組みである。

### 7.2 基本ルール

プレイヤーは、お題に対して手持ちカードを選び、関係線を作る。

正しい関係でつながると、町の出来事の背景が見える。

### 7.3 Chapter 1の例

お題：

> 寺の人が市へ取りに来なかった理由を考えよう。

候補カード：

- 市の荷物
- 供物
- 僧
- お寺の瓦
- 寺をつくる職人たち
- 木簡
- 役人

正しい接続例：

- 市の荷物 + 供物 + 僧
- お寺の瓦 + 寺をつくる職人たち + 飛鳥寺
- 木簡 + 役人 + 市の荷物

読み解き例：

> 寺では儀式準備と建設作業が重なり、物資の受け取りが混乱していた。

### 7.4 失敗時

間違ったカードを選んでもゲームオーバーにしない。

うーちゃんや記録帳が、やさしく不足を示す。

例：

> うーちゃん：「このカードも関係ありそうだけど、まだ何か足りないかも！」

## 8. Chapter 1の勝利条件

### 8.1 章のゴール

Chapter 1のゴールは、飛鳥寺に関わるカードを集め、寺で起きている混乱の理由をカードで読み解き、Sランク「飛鳥寺」カードを解放すること。

### 8.2 クリア条件案

以下を満たすとChapter 1クリア。

1. 必須カードを一定数集める
2. 寺へ向かう
3. カード接続パズルを解く
4. 寺で起きていた問題を理解する
5. Sランク「飛鳥寺」カードを解放する
6. 次時代への扉、または時代横断リンクが開く

### 8.3 必須カード候補

- 市の荷物
- 供物
- お寺の瓦
- 寺をつくる職人たち
- 木簡
- 役人
- 僧

### 8.4 Sランク解放

Sランク「飛鳥寺」は、寺に到着するだけでは完全解放されない。

以下の関連カードに触れることで、カードの輪郭が完成していく。

- お寺の瓦
- 寺をつくる職人たち
- 僧
- 供物
- 木簡または役人

## 9. MVPカード枚数

### 9.1 基本方針

飛鳥パック全体は将来的に24〜32枚でもよい。

ただし、MVPでは12〜16枚に絞る。

### 9.2 理由

- 実装負荷を下げる
- 史実調査の精度を高める
- 最初のカードループが面白いか検証しやすい
- カード1枚ごとの意味を薄めない

### 9.3 MVP配分案

| 種類 | 枚数 |
|---|---:|
| Sランク | 1〜2 |
| Aランク | 5〜6 |
| Bランク | 5〜8 |
| Cランク | 0〜2 |

### 9.4 初期MVPカード候補

#### Sランク

- 飛鳥寺
- 飛鳥大仏、または蘇我馬子は後続候補

#### Aランク

- お寺の瓦
- 寺をつくる職人たち
- 木簡
- 役人
- 僧
- 供物

#### Bランク

- 市の荷物
- 木材を運ぶ人々
- 素朴な器
- 工人
- 宮に仕える人
- 僧の衣

#### 保留

- 甘い香りの屋台
- 飛鳥のお菓子
- 子どもの遊び
- 蜜

## 10. カード入手演出

### 10.1 基本方針

派手な演出より、以下を重視する。

- 見つけた感
- 記録された感
- 関連がつながった感
- Sカードに近づいた感

### 10.2 入手演出例

通常カード：

> 「お寺の瓦」を見つけた！  
> 関連：寺をつくる職人たち  
> Sカード「大きなお寺のカード」の手がかりが増えた。

複数収集カード：

> 「木簡」をもう一つ見つけた！  
> 発見数：2 / 3  
> 新しい発見：荷物にも木の札が使われている。

Sカード解放：

> Sランクカード「飛鳥寺」が記録帳に刻まれた。  
> たくさんの人・物・技術・信仰が集まって、このお寺が作られている。

## 11. カード帳UI

### 11.1 必須画面

MVPで必要なカード帳画面は以下。

- 時代パック一覧
- 飛鳥パック詳細
- カード詳細
- 未発見カード枠
- 関連カードリンク
- セット進捗
- Sカードのシルエット
- 絵柄違い一覧
- 後から追加された時代リンク表示

### 11.2 最重要画面

最重要は、カード同士が線でつながる画面。

カードがただの図鑑ではなく、時代構造を表すネットワークとして見える必要がある。

### 11.3 UI例

```text
飛鳥パック 7 / 14

S  0 / 1
A  4 / 6
B  3 / 7

[市の荷物] ─ [木簡] ─ [役人]
     │
  [供物] ─ [僧] ─ [未発見S]

[お寺の瓦] ─ [寺をつくる職人たち] ─ [未発見S]
```

## 12. 最初の5分のプレイフロー

### 12.1 目的

最初の5分で、プレイヤーにこのゲームの面白さを伝える。

最低限、以下を体験させる。

- うーちゃんとの出会い
- 飛鳥の町への移動
- 最初の史実カード獲得
- カード帳の確認
- 未発見カードと総数の提示
- Sカードのシルエット提示

### 12.2 流れ

1. 寝る前の絵本シーン
2. 時の回廊へ入る
3. うーちゃんとぶつかる
4. 名前入力
5. 飛鳥の町へ入る
6. 市の一角で寺向けの荷を見る
7. 最初のカード「市の荷物」または「供物」を獲得
8. カード帳が開く
9. 飛鳥パック 1 / 14 が表示される
10. 未発見Sカードのシルエットが見える
11. 寺の人が来ないことを知る
12. 寺へ向かう目的が生まれる

### 12.3 注意点

- 最初の5分で説明を長くしない
- カード帳を必ず一度見せる
- 史実性の強いカードから始める
- 甘い屋台など、確度が弱いものは初回導入から外す

## 13. 史実調査パイプライン

### 13.1 必要性

この企画は、史実性が屋台骨である。

そのため、カード1枚ごとに小さな時代考証が必要である。

### 13.2 カードごとの必須項目

- カード名
- カテゴリ
- レアリティ
- 史実確度
- 史実根拠
- 参考資料
- 採用理由
- 子ども向け表現
- 避ける表現
- 関連カード
- 時代横断候補
- ゲーム内の入手条件

### 13.3 史実確認の優先順位

1. Sランクカード
2. Chapter 1必須カード
3. カード接続パズルに使うカード
4. 絵柄違いカード
5. Bランク背景カード
6. Cランク演出候補

## 14. 実装フェーズ

### Phase 1：カード収集MVP

- 飛鳥パック 12〜16枚
- カード獲得
- カード帳
- Sカードシルエット
- 基本的な関連カード表示

### Phase 2：カード接続パズル

- Chapter 1の終盤に導入
- カードを選んで関係を作る
- 寺で起きていた混乱を読み解く

### Phase 3：再訪・絵柄違い

- 発見場所違い
- 複数収集カード
- カード完成度
- 再訪時の追加会話

### Phase 4：時代横断リンク

- 奈良以降で過去カードにリンク追加
- テーマパック拡張
- 過去カードの再解釈

### Phase 5：対戦・AI対戦

- 構造バトル
- テーマ対戦
- 時代横断対戦

## 15. 成功条件

この仕様が成功している状態。

- ストーリーを進めるだけでカードが自然に集まる
- カード帳を開くのが楽しい
- 未発見カードを探したくなる
- Sカードを解放したくなる
- カード接続で町の出来事が理解できる
- 史実性が崩れていない
- 子どもでも直感的に遊べる
- 大人が見ても歴史構造が面白い

## 16. 失敗条件

以下は避ける。

- ただの図鑑になる
- ただの会話ADVになる
- ただのカード収集作業になる
- クイズになる
- Sカードだけを集めればよくなる
- 史実性より演出が優先される
- カード接続が雑で誤解を生む
- UIが複雑すぎて子どもが触れない

## 17. 最重要結論

> Learning RPGのゲーム性は、カードを集めることではなく、カードで時代を読み解くことにある。

Chapter 1 MVPでは、まず以下を成立させる。

1. 飛鳥の町に入る
2. 史実カードを集める
3. カード帳で関係を見る
4. カード接続パズルで町の出来事を読み解く
5. Sランク「飛鳥寺」を解放する

ここまでできれば、Learning RPG は単なる企画ではなく、ゲームとして動き始める。
