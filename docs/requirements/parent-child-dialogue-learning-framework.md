# Parent-Child Dialogue Learning Framework

## 位置づけ

このドキュメントは、`learning-rpg` の将来構想として、親子対話・判断ログ・学びの可視化をRPG体験に有機的に埋め込むためのタスク案である。

現時点では本格実装しない。まずはプロダクトラボ寄りの構想・設計メモとして保持し、RPG本体の基本体験が安定した後に段階的に検証する。

## コンセプト

親子で一緒に遊びながら、子どもが何を見て、どう考え、どんな判断をしたのかを、ゲーム内の選択と対話メモとして残す。

学習教材をゲームに貼り付けるのではなく、冒険、NPC会話、選択肢、戦闘、キャンプファイア会話、振り返りの中に学びを自然に埋め込む。

表側の体験名候補：

- こころの冒険日誌
- 親子対話クエスト
- Learning Quest System

裏側の技術名候補：

- Choice Memory System
- Parent-Child Dialogue Log
- Dialogue Quest Engine

## 目指す体験

- 子どもが、冒険の中で自分なりに考える
- 親が、答えを教えるのではなく問いを返せる
- 「親に聞く」という選択肢を自然なゲームコマンドとして扱う
- 子どもの選択を正解/不正解で採点しない
- 選択傾向を、成長の記録として後から見返せる
- 親自身も、自分がどう問いかけ、どう受け止めたかを振り返れる
- 個人OS側では、将来的にRPG学びログ・親子対話ログとして参照できる

## なぜ価値があるか

親子向けアプリは、家族共有、写真共有、見守り、教育コンテンツ、育児アドバイス、学校連絡などが多い。
一方で、親と子どもが一緒に遊びながら、対話・判断・感情・価値観をログ化して育てるアプリはまだ少ない可能性が高い。

この構想の価値は、以下の掛け合わせにある。

- RPGの冒険体験
- 親子のリアルな会話
- 子どもの判断ログ
- 親の問いかけログ
- 学習テーマの段階設計
- 個人OS/プロダクトラボとの連携

## 参考市場メモ

2026-05-16時点の初期調査では、近い領域は存在するが、完全一致するプロダクトは見つかっていない。

近いカテゴリ：

1. 子ども向けメッセージ/家族連絡
   - Messenger Kids
   - myFirst Circle
   - Apple/Discordなどのペアレンタル管理系

2. 家族共有/祖父母共有
   - 写真共有、家族SNS、孫との近況共有系
   - 祖父母と孫の会話支援に関する研究も出てきている

3. 育児支援/発達記録
   - Daily Growth系の育児アドバイス
   - 発達記録、生活ログ、子育てTips

4. 教育コンテンツ/読み聞かせ支援
   - StoryBuddy
   - ContextQ
   - eaSEL
   - Captivate!

5. 見守り/制限/監視
   - Parental control系
   - スクリーンタイム、Webフィルタ、位置情報、利用状況確認

差別化余地：

- 親子の会話そのものを主役にする
- 子どもだけでなく親の問いかけも記録する
- 採点ではなく、判断傾向と成長のログにする
- RPG/物語/選択肢をトリガーにして会話を生む
- 個人OSに接続し、家庭内の学びのナレッジベースに育てる

## コア機能案

### 1. 親に聞くコマンド

選択肢の1つとして「お父さんに聞く」「お母さんに聞く」「家族に相談する」を用意する。

例：

```text
森の主は怒っている。
でも、なぜ怒っているのかはまだ分からない。

どうする？
1. 剣をかまえる
2. 少し離れて様子を見る
3. 森の主に話しかける
4. お父さんに聞いてみる
```

### 2. 対話モード

「親に聞く」を選ぶと、ゲームを一時停止し、親子で話す画面に切り替える。

```text
お父さんに聞いてみよう。

質問：
森の主は、どうして怒っていると思う？

子どもの言葉メモ：
[ 入力欄 ]

親の気づきメモ：
[ 入力欄 ]
```

### 3. 選択肢タグ

各選択肢に判断タイプを付与し、選択ログとして保持する。

判断タイプ例：

- observation: まず見る
- dialogue: 話してみる
- direct_action: まず行動する
- collaboration: 誰かに相談する
- help_seeking: 助けを求める
- empathy: 相手の気持ちを考える
- systems_thinking: 全体のつながりを見る
- risk_avoidance: 無理をしない
- creativity: 新しい方法を考える
- rule_oriented: ルールや約束を確認する

### 4. 選択ログ

```json
{
  "timestamp": "2026-05-16T14:30:00+09:00",
  "player": "child",
  "scene": "first_forest_before_boss",
  "questionId": "forest_guardian_choice_001",
  "selectedChoice": "ask_parent",
  "choiceTags": ["collaboration", "help_seeking", "parent_child_dialogue"],
  "childComment": "木を切ったから怒っていると思う",
  "parentMemo": "怒りの理由を一緒に考えられた。倒す以外の選択肢に気づいていた。"
}
```

### 5. キャンプファイア会話

冒険の区切りに、親子で短く振り返る。

問いの例：

- 今日いちばん面白かったところは？
- どの選択がむずかしかった？
- どのキャラクターの気持ちが気になった？
- 現実の世界にも似たことはある？
- 次はどうしてみたい？

## 子ども向けの扱い方

子どもを分析対象として扱いすぎない。

避ける表現：

- 共感力が低い
- 判断力が弱い
- 短期志向が強い

望ましい表現：

- 今回はすぐ行動する選択が多かった
- 相手の理由を考える発言が出てきた
- 困ったときに相談する選択ができた
- 観察してから動く選択が少し増えた

目的は評価ではなく、成長の記録と親子の対話を増やすこと。

## 将来フェーズ

### Phase 0: 構想保持

- このドキュメントを設計メモとして保持
- すぐには本格実装しない
- RPG本体の基本体験を優先する

### Phase 1: 静的な対話イベント

- 1面の森エリアに親子対話イベントを1〜3個だけ追加
- まだログ保存は必須にしない
- まずはゲーム体験を壊さないか確認する

### Phase 2: 選択タグの導入

- イベント選択肢にタグを付与
- ローカルに選択履歴を保存
- UIには分析結果を出しすぎない

### Phase 3: 親子対話メモ

- 「親に聞く」選択肢を追加
- 子どもの言葉メモ、親の気づきメモを残せるようにする
- 冒険日誌として見返せるようにする

### Phase 4: 個人OS連携

- RPG側の学びログを個人OS/プロダクトラボから参照
- 親子対話ログ、学習テーマ、判断タグをまとめて表示
- 家族向け画面と親向け分析画面を分ける

### Phase 5: 独立プロダクト検証

- RPGから切り出し、親子対話アプリ単体として成立するか検証
- 家庭内の対話カード、物語生成、感情ログ、判断ログを扱う
- 祖父母との共有、夫婦間の育児観共有は拡張機能として検討

## 実装時の注意

- 子どもの個人データを扱うため、初期はローカル保存を優先する
- 外部送信やクラウド同期は後回しにする
- 親のための分析と、子どもに見せる冒険日誌は分離する
- 子どもには評価スコアを見せない
- AIを使う場合も、親子の対話を置き換えるのではなく、問いを支援する役割にする
- ゲームテンポを壊さないよう、対話イベントは短く、任意でスキップ可能にする

## 完了条件案

将来タスクとして着手する場合の最小完了条件：

- 1面の森エリアに対話イベントが最低1つ存在する
- 「親に聞く」選択肢が1つ存在する
- 選択肢に判断タグが付いている
- 選択結果をローカル保存できる
- 子どもの言葉メモと親メモを保存できる
- 冒険後に簡易の振り返りが表示される
- 採点ではなく、成長記録として表示される

## 現時点の判断

この構想は、通常の学習RPGよりもプロダクトラボ色が強い。
ただし、`learning-rpg` の個性を大きく伸ばせる可能性があるため、将来像として保持する価値が高い。

短期ではRPG本体の基本ループを優先し、中期以降で親子対話ログ機能を小さく検証する。
