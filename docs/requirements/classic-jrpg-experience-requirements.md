# Classic JRPG Experience Requirements

## 目的

現在のRPGプロトタイプは、まだ「遊べるRPG体験」としては初期段階にある。
次の開発では、見た目や演出の作り込みよりも、プレイヤーが気持ちよく前に進める基本体験を成立させることを最優先にする。

この要件書は、機能一覧ではなく、**プレイヤーがどう気持ちよく進むか**を中心に設計するためのものとする。

## 目指す体験

本作は、クラシックなコマンド選択式JRPGの基本体験を持つブラウザRPGを目指す。

重視するのは、既存有名RPGの固有名詞・UI・音楽・キャラクターを模倣することではなく、以下のような体験構造である。

- 町で情報を集める
- フィールドへ出る
- 敵と遭遇する
- コマンドで戦う
- 経験値とゴールドを得る
- レベルアップや装備更新で強くなる
- 町に戻って回復する
- もう少し遠くへ行きたくなる

派手さよりも、シンプルで分かりやすく、少しずつ前に進みたくなるテンポを優先する。

---

## 今回の開発ゴール

プレイヤーが5〜10分間、明確な目的を持って遊べる状態にする。

最低限、以下の流れを自然に体験できること。

1. タイトル画面からゲームを開始する
2. 町にいる
3. NPCと話して目的を知る
4. フィールドに出る
5. 敵と遭遇する
6. コマンド式バトルで戦う
7. 勝利して経験値とゴールドを得る
8. レベルアップする
9. HP/MPが減る
10. 町に戻って回復する
11. ゴールドで装備を更新する
12. 強くなった状態で再び探索に出る
13. 最初の小目標を達成する

## 体験設計

### プレイヤーに感じてほしいこと

- 最初は弱い
- 何度か戦うと少し強くなる
- 装備を買うと戦闘が楽になる
- 無理をすると負ける
- 町に戻ると安心できる
- 次の目的地へ進みたくなる

### 重要な体験キーワード

- 安心できる町
- 少し怖いフィールド
- 分かりやすい戦闘
- 数値で分かる成長
- 町に戻る判断
- 次の目的地への誘導

---

## コアループ

プレイヤーは以下を繰り返す。

1. 町で準備する
2. NPCからヒントを得る
3. フィールドに出る
4. 敵と戦う
5. 経験値とゴールドを得る
6. HP/MPが減る
7. 町に戻る
8. 回復・装備更新する
9. より遠くへ進む

### ループ成立条件

以下が成立していれば、最低限RPGとして遊べる。

- 敵を倒すメリットがある
- 戦闘でリスクがある
- 回復する意味がある
- ゴールドを使う先がある
- レベルアップで明確に強くなる
- 次に行く場所が分かる

## 必須画面

### 1. タイトル画面

目的：ゲーム開始前の入口を作る。

要件：

- ゲームタイトルを表示する
- 「はじめる」ボタンを表示する
- セーブ機能がある場合のみ「つづきから」を表示する
- まずは「はじめる」のみでもよい

完了条件：

- ボタンを押すと町画面へ遷移する

### 2. 町画面

目的：プレイヤーの安全地帯・拠点にする。

要件：

- NPCが最低3人いる
- 宿屋または回復ポイントがある
- 武器屋または装備更新ポイントがある
- フィールドへ移動できる出口がある
- プレイヤーのHP/MP/GOLDを確認できる

町に必要な要素：

- NPC A：目的地を教える
- NPC B：戦闘や回復のヒントを教える
- NPC C：世界観や雰囲気を伝える
- 宿屋：HP/MPを全回復する
- 店：武器または防具を買える
- 出口：フィールドへ移動する

完了条件：

- NPCに話しかけられる
- 宿屋で回復できる
- 店で装備を買える
- フィールドへ移動できる

### 3. フィールド画面

目的：探索と戦闘の場にする。

要件：

- プレイヤーが上下左右に移動できる
- 一定歩数または確率で敵と遭遇する
- 町に戻れる
- 最初の目的地が存在する
- HPが減った状態で探索を続けるリスクがある

フィールド要素：

- 町への入口
- 草原エリア
- 森エリア
- 小目的地
- 敵エンカウント

完了条件：

- 移動できる
- 敵と遭遇する
- 町に戻れる
- 目的地に到達できる

### 4. 戦闘画面

目的：クラシックJRPGらしいコマンド戦闘を成立させる。

要件：

- プレイヤーと敵の状態を表示する
- コマンドを選択できる
- 戦闘ログを表示する
- 勝敗がある
- 勝利時に報酬がある
- 敗北時に町へ戻される

表示項目：

- プレイヤーHP/MP
- 敵名
- 敵HP
- コマンド
- 戦闘ログ

コマンド：

- たたかう
- じゅもん
- どうぐ
- にげる

完了条件：

- コマンドで戦える
- 敵を倒せる
- 敵からダメージを受ける
- 勝利報酬を得られる
- 敗北時に町へ戻る

### 5. ステータス画面

目的：成長を見える化する。

要件：

- プレイヤーの現在ステータスを表示する
- 装備を表示する
- 次のレベルまでの経験値を表示する

表示項目：

- 名前
- レベル
- HP
- MP
- 攻撃力
- 防御力
- 経験値
- 次のレベルまで
- ゴールド
- 武器
- 防具

完了条件：

- 現在の強さが分かる
- レベルアップ後に数値変化が分かる

```ts
type Player = {

  level: number;

  maxHp: number;
  mp: number;
  maxMp: number;

  defense: number;
  exp: number;
  gold: number;
  weaponId: string | null;
  armorId: string | null;

```

初期値の目安：

```ts
const initialPlayer = {
  name: "Hero",
  level: 1,
  hp: 24,
  maxHp: 24,

  maxMp: 6,

  armorId: "cloth_armor",
  items: [{ itemId: "herb", count: 2 }]

```

### 成長方針

- レベル1では弱い敵にも少し苦戦する
- レベル2になると弱い敵が楽になる
- レベル3になると普通の敵にも安定して勝てる
- 装備を更新すると明確に被ダメージや与ダメージが変わる

### レベルテーブルの目安

```ts
const levelTable = [
  { level: 1, requiredExp: 0 },
  { level: 2, requiredExp: 12 },
  { level: 3, requiredExp: 32 },
  { level: 4, requiredExp: 65 },
  { level: 5, requiredExp: 110 }
];
```

レベルアップ時の成長：

- maxHp +5〜8
- maxMp +2〜4
- attack +2〜3
- defense +1〜2
- HP/MPを全回復してもよい

## 戦闘仕様

### 基本ルール

- ターン制
- プレイヤーが先に行動する
- プレイヤー行動後、敵が生きていれば敵が行動する
- どちらかのHPが0になるまで繰り返す

### ダメージ計算

```ts
damage = Math.max(1, attacker.attack - defender.defense + random(-2, 2));
```

要件：

- 最低1ダメージは入る
- ランダム幅を少し入れる
- クリティカルは後回しでよい

### じゅもん

最初は2種類でよい。

```ts
const spells = [
  {
    id: "fire",
    name: "火の玉",
    mpCost: 3,
    effect: "enemy_damage",
    power: 12
  },
  {
    id: "heal",
    name: "小回復",
    mpCost: 3,
    effect: "self_heal",
    power: 18
  }
];
```

要件：

- MPが足りない場合は使えない
- 攻撃魔法は通常攻撃より強い
- 回復魔法は探索継続の判断材料になる

### どうぐ

初期アイテム：

```ts
const items = [
  {
    id: "herb",
    name: "薬草",
    effect: "heal",
    power: 20,
    price: 10
  }
];
```

要件：

- 薬草を使うとHPが回復する
- 使うと個数が減る
- 個数が0なら使えない

### にげる

要件：

- 成功率は70%
- 成功時はフィールドに戻る
- 失敗時は敵の攻撃を受ける

### 戦闘ログ

ログは短文で表示する。

例：

- Heroのこうげき！
- 敵に5のダメージ！
- 敵のこうげき！
- Heroは3のダメージをうけた！
- 敵をたおした！
- 経験値4を手に入れた！
- 8ゴールドを手に入れた！

### 勝利処理

- 経験値を得る
- ゴールドを得る
- レベルアップ判定をする
- フィールドへ戻る

### 敗北処理

- 町に戻る
- HPを最大値の半分まで回復する
- ゴールドを減らす処理は今回は不要
- 「目を覚ますと町に戻っていた」などの簡単なログを表示する

## 町・NPC仕様

NPCは単なる飾りではなく、プレイヤーの行動を誘導する。

### NPC A：目的提示

例：

> 北の森で、あやしい光を見た人がいるらしい。

目的：次の目的地を示す。

### NPC B：戦闘ヒント

例：

> 傷ついたら無理をするな。町に戻って休むんだ。

目的：回復の重要性を伝える。

### NPC C：装備ヒント

例：

> 森に行くなら、少し良い武器を持っていくと安心だよ。

目的：ゴールドを使う理由を伝える。

### NPC D：世界観

例：

> この町は旅人たちの休み場さ。外は魔物が多いけどね。

目的：世界観を少しだけ出す。

### NPC会話要件

- 1会話は1〜3文まで
- 1文は短くする
- 次に何をすればいいか分かる内容を優先する
- 難解な設定説明は避ける

## フィールド・探索仕様

### 移動

- 上下左右に移動できる
- キーボードまたは画面ボタンで操作できる
- スマホ対応はP1でよいが、最低限クリック・タップ操作も考慮する

### エンカウント

方式：

- 一定歩数ごとに確率判定する
- 例：5歩以降、1歩ごとに20%でエンカウント

```ts
if (stepsSinceBattle >= 5 && Math.random() < 0.2) {
  startBattle();
}
```

### エリア

最初は2エリアでよい。

#### 草原

- 町の近く
- 弱い敵が出る
- 初心者向け

#### 森

- 少し遠い
- 強めの敵が出る
- 小目的地がある

## 最初のミニクエスト

### クエスト名

北の森の光

### 目的

プレイヤーに最初の明確なゴールを与える。

### 流れ

1. 町のNPCが北の森の異変を話す
2. プレイヤーがフィールドに出る
3. 草原で敵と戦って成長する
4. ゴールドを貯めて装備を更新する
5. 森に進む
6. 森の奥で強めの敵と戦う
7. 勝利後、光を調べる
8. 町に戻る
9. NPCから感謝される
10. 「プロトタイプはここまで」と表示する

### クリア条件

- 森の奥のイベントを確認する
- または強い敵を倒す

### 報酬

- 経験値
- ゴールド
- 簡単な称号またはメッセージ

例：

> 小さな冒険者として、一歩を踏み出した。

## UI / UX方針

### 基本方針

- 画面はシンプルにする
- 次に押せるボタンを分かりやすくする
- 状態変化を必ず見せる
- ログを使って行動結果を明確に伝える

### 常時表示したい情報

- HP
- MP
- Lv
- Gold
- 現在地

### 戦闘UI

優先順位：

1. コマンドが押しやすい
2. HP変化が分かる
3. ログが読みやすい
4. 敵の状態が分かる

### フィールドUI

優先順位：

1. どこにいるか分かる
2. 町に戻れることが分かる
3. 次の目的地がうっすら分かる
4. 移動操作が迷わない

## バランス調整方針

### プレイ時間

最初のプロトタイプは5〜10分で一区切りつくようにする。

- 最初の2〜3戦でレベル2になる
- さらに3〜5戦でレベル3になる

- 弱い敵を5〜7体倒すと最初の装備が買える

### 戦闘難易度

- 町の近くでは負けにくい
- 森では油断すると負ける
- 回復や装備更新をすると安定する

## Codexへの実装指示

既存コードを確認し、現在の構造をできるだけ活かしながら、P0要件を満たす最小実装を行う。

優先するのは見た目の完成度ではなく、ゲームループの成立である。

実装順序：

1. 現在の画面構成とコード構造を確認する
2. プレイヤーデータを定義する
3. 町画面を作る
4. NPC会話を実装する
5. フィールド移動を実装する
6. 敵エンカウントを実装する
7. バトル画面を実装する
8. 経験値・ゴールド・レベルアップを実装する
9. 宿屋・回復処理を実装する
10. 店・装備更新を実装する
11. 最初のミニクエストを実装する
12. READMEまたは開発メモに変更内容を記録する

実装時の注意：

- 既存の有名RPGの固有表現は直接使わない
- UIはシンプルでよい
- まず遊べることを優先する
- バランス値は後から調整しやすいようにデータ化する
- ゲームロジックとUIをできるだけ分ける
- エラーが出ない状態でコミットできるようにする

実装後に記録すること：

- 追加した機能
- 変更したファイル
- 操作方法
- 現在できること
- まだ未実装のこと
- 次に改善すべきこと
